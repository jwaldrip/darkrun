darkrun — desktop shell

A native desktop layout (not a centered web page): a unified toolbar, a left sidebar of projects with runs nested under each, a Mine / All filter that scopes runs to the ones you authored (by commit author), and a main pane that fills the window. Mobile collapses the sidebar to a drawer. Shown in dark and light — the app + site follow the system theme.

project = repo, runs = worktrees   A project is a git repo (not an epic) — its name comes from the repo directory or the name in .darkrun/settings.yml. The items nested under it are the repo's darkrun worktrees / runs: each run is a git worktree on a darkrun/*/* branch. Branches or worktrees that don't match the darkrun/*/* scheme are NOT darkrun runs and are hidden from the list.
new run   The ＋ new run… entry under a repo lets you start a run from the UI — it surfaces the start command to paste into your harness (per-harness, like the no-engine view), so you can kick off a new darkrun/*/* worktree without leaving the app.
Mine filter   Mine / All scopes the run list to runs you authored — matched on the commit author of each run's branch against your git identity (user.email). "All" shows the whole team's runs on shared projects. The me tag marks yours. Open: do we match on any commit by you, or only runs where you're the primary author?
native feel + theming   Full-height sidebar + unified toolbar (no centered web hero), list density tuned for desktop, content fills the window and is resizable. The app reads the system appearance (prefers-color-scheme) and themes light/dark to match; the website does the same. Light derives a higher-contrast accent (#0e9fd6) since the dark cyan is too light on white.
