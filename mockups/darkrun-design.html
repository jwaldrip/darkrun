08 · Architecture

The model underneath

One method, many lines

The phase machine is universal — every station in every factory walks spec → review → manufacture → audit → reflect → checkpoint. The stations are per-factory (the software factory's six are Frame→Harden, declared in FACTORY.md). The product pitch is the method + the cost-of-late-discovery principle; each factory instantiates its own line.

Connect, don't show

The permission boundary is the checkpoint, not per-tool prompts — so harnesses launch autonomous by default (per-harness flag, toggle to dial back). On boot the engine binds an ephemeral port and writes ~/.darkrun/<slug>/engine-<pid>.json; the desktop discovers it and auto-connects. No show, no env port.

Annotation = anchor + expression + intent

Version-pinned, stored at .darkrun/<run>/annotations/, retained for debuggability. Text re-anchors by quote on file change; image crops the region; html resolves pixels to file:line. Severity (must/should/nit) steers the checkpoint. The human points freely; the agent gets a precise source target + intent.

Distribution

End-to-end Rust. npm per-arch binaries; the desktop is Dioxus, launched via LaunchServices with the embedded .icns. Licensed FSL-1.1-ALv2 (Apache-2.0 after two years). Seven harnesses: Claude Code, Codex, Gemini CLI, Cursor, Windsurf, OpenCode, Kiro.
